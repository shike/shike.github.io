# GitHub 个人主页设计文档

## 项目概述

为施可（Shi Ke）构建一个部署在 GitHub Pages 上的个人主页。单页滚动式设计，专业商务风格，中英双语支持。

---

## 需求确认

| 需求项 | 确认内容 |
|--------|----------|
| 展示内容 | 个人简介、技能栈、履历时间线、演讲与发表、项目展示、联系方式 |
| 页面风格 | 专业商务风（干净排版、中性色调、简历感） |
| 页面结构 | 单页滚动式 |
| 技术栈 | 纯 HTML / CSS / JS（零依赖） |
| 语言 | 中英双语切换 |
| 部署目标 | GitHub Pages |

---

## 页面结构

共 8 个区块，从上到下依次排列：

1. **固定导航栏** — Logo(施可) + 导航链接 + 中英切换按钮
2. **Hero 区域** — 姓名、职位、一句话定位
3. **关于我 (About)** — 个人简介、慢闪店理念
4. **技能栈 (Skills)** — 核心能力标签展示
5. **履历时间线 (Experience)** — 工作经历纵向排列
6. **演讲与发表 (Speaking)** — 峰会演讲、专栏文章
7. **项目展示 (Projects)** — 可计算开店科技等项目卡片
8. **联系方式 (Contact) + 页脚** — 邮箱、社交媒体、版权信息

## 视觉设计规范

### 配色方案

| 用途 | 色值 |
|------|------|
| 页面背景 | `#FFFFFF` |
| 区块交替背景 | `#F9FAFB` |
| 导航栏背景 | `#111827` |
| 主标题 / 正文 | `#111827` |
| 副标题 / 次要文字 | `#6B7280` |
| 链接、按钮、高亮 | `#2563EB` |
| 分割线 / 边框 | `#E5E7EB` |

### 字体排版

- **字体栈**：`-apple-system, "PingFang SC", "Microsoft YaHei", sans-serif`
- **Hero 姓名**：48px / font-weight: 700
- **区块标题**：28px / font-weight: 600，附英文副标题 12px 全大写追踪字距
- **正文**：16px / line-height: 1.75 / color: `#374151`
- **时间线年份**：14px / font-weight: 600 / color: `#2563EB`

### 间距规范

- 页面最大宽度：1100px，居中
- 每个区块内边距：上下 80px
- 奇偶区块背景色交替（白 / `#F9FAFB`）
- 卡片间距：24px
- 移动端：内边距缩减至 40px，字号等比例缩小

## 技术架构

### 文件结构

```
github-homepage/
├── index.html          # 主页面
├── css/
│   └── style.css       # 全部样式
├── js/
│   └── main.js         # 交互逻辑
└── assets/
    └── avatar.jpg      # 头像（可选）
```

### 核心交互

| 功能 | 实现方式 |
|------|----------|
| 导航高亮 | `IntersectionObserver` 监听各区块进入视口 |
| 平滑滚动 | `element.scrollIntoView({ behavior: 'smooth' })` |
| 中英切换 | 所有文本双份写入 `data-zh` / `data-en` 属性，JS 批量替换 `textContent` |
| 移动端菜单 | 汉堡按钮切换导航显示/隐藏 |

### 响应式断点

- **桌面端**：> 768px，完整导航 + 多列布局
- **移动端**：<= 768px，汉堡菜单 + 单列堆叠

## 部署方案

1. GitHub 新建仓库 `shike.github.io`（或任意仓库名 + 自定义域名）
2. 推送代码到 `main` 分支
3. Settings → Pages → 选择 `main` 分支 → 保存
4. 约 1-2 分钟后可通过 `https://shike.github.io` 访问
